# СТПО — Спецификация требований к программному обеспечению «PySandbox Edu»

## 1. Введение

- **Название системы**: PySandbox Edu.
- **Назначение**: учебная онлайн-песочница для безопасного запуска программ на языке Python в изолированных Docker-контейнерах.
- **Область применения**: лабораторные работы, практикумы и демонстрации по дисциплинам, связанным с программированием и DevOps.

## 2. Требования к функциям

- **F1. Запуск кода**: система должна принимать Python-код от пользователя и выполнять его.
- **F2. Изоляция**: каждый запуск кода выполняется в отдельном контейнере на основе образа `pysandbox-executor`.
- **F3. Лимиты ресурсов**: при выполнении кода должны применяться ограничения по памяти и CPU.
- **F4. Таймаут**: выполнение кода должно прерываться по таймауту (например, 10 секунд).
- **F5. Логирование**: система должна логировать входящие запросы, результаты выполнения кода и ошибки.
- **F6. Веб-интерфейс**: пользователь должен иметь возможность ввести код и увидеть результаты через браузер.
- **F7. API**: должен быть документированный REST API для интеграции с внешними системами.

## 3. Нефункциональные требования

- **Безопасность**:
  - пользовательский код не должен иметь доступа к хост-системе;
  - сеть для контейнеров с кодом должна быть отключена;
  - рекомендуется запускать код от непривилегированного пользователя внутри контейнера.
- **Производительность**:
  - время старта контейнера и выполнения простых программ не должно превышать нескольких секунд;
  - система должна корректно обрабатывать несколько последовательных запусков кода.
- **Надёжность**:
  - в случае ошибки Docker-сервера или отсутствия образа должна возвращаться понятная ошибка API;
  - при падении контейнера ресурсы должны освобождаться автоматически.
- **Удобство использования**:
  - понятный интерфейс с выделением stdout/stderr и кодом возврата;
  - понятные сообщения об ошибках (в том числе при превышении лимитов).

## 4. Ограничения

- Максимальная длина кода: 10 000 символов.
- Память контейнера: по умолчанию 128 MB.
- CPU: не более ~0.5 CPU (конфигурируемо).
- Сеть: отключена для контейнеров с кодом.

## 5. Взаимодействие с внешними системами

- Docker Engine через сокет `/var/run/docker.sock` (монтируется в контейнер backend).
- Браузер пользователя через HTTP(S).

## 6. Требования к тестированию

- Должны быть разработаны тест-планы и тест-кейсы для:
  - функциональности запуска кода;
  - проверки ограничений (таймаут, память, длина кода);
  - обработки ошибок Docker;
  - проверок CORS и доступности API с frontend.

Подробнее см. `Тестирование.md`.
