# Архитектура системы «PySandbox Edu»

## 1. Общий обзор

Система состоит из трёх основных компонент:

- **Frontend** — статический веб-интерфейс (HTML/CSS/JS), отдаётся через nginx.
- **Backend** — REST API на FastAPI, осуществляет приём кода и управление Docker-контейнерами.
- **Executor** — Docker-образ с Python 3.10, внутри которого выполняется пользовательский код.

Все компоненты разворачиваются и оркестрируются через `docker-compose`.

## 2. UML: Диаграмма компонентов (текстовое описание)

Рекомендуемая диаграмма (можно нарисовать в любом UML-инструменте):

- Компонент `Web Browser`:
  - взаимодействует с компонентом `Frontend (nginx)` по HTTP (порт 3000).
- Компонент `Frontend (nginx)`:
  - отдаёт статические файлы `index.html`, `styles.css`, `script.js`;
  - взаимодействует с компонентом `Backend (FastAPI)` по HTTP (порт 8000).
- Компонент `Backend (FastAPI)`:
  - предоставляет REST API `/`, `/health`, `/execute`;
  - использует компонент `Docker Engine` через сокет `/var/run/docker.sock`.
- Компонент `Docker Engine`:
  - создаёт и управляет контейнерами на основе образа `pysandbox-executor`.

Добавьте графическую UML-диаграмму компонентов в отчёт, используя это описание.

## 3. UML: Последовательность (Sequence) при выполнении кода

Участники:

- `User`
- `Browser`
- `Frontend`
- `Backend`
- `Docker Engine`
- `Executor Container`

Основной сценарий:

1. `User` вводит код и нажимает «Запустить» в `Browser`.
2. `Browser` отправляет запрос `POST /execute` на `Frontend` (nginx, проксирование) или напрямую на `Backend` (в текущей реализации — напрямую).
3. `Frontend`/`Browser` → `Backend`: HTTP-запрос с JSON `{ code: "..." }`.
4. `Backend` валидирует длину кода и формирует команду для запуска Python внутри контейнера.
5. `Backend` → `Docker Engine`: запрос на создание и запуск контейнера из образа `pysandbox-executor` с ограничениями по памяти/CPU, отключённой сетью и автоудалением.
6. `Docker Engine` создаёт `Executor Container` и запускает в нём интерпретатор Python с переданным кодом.
7. `Executor Container` выполняет код и отдаёт stdout/stderr и код возврата обратно в `Docker Engine`.
8. `Docker Engine` завершает контейнер и удаляет его (auto-remove).
9. `Backend` получает результат, формирует ответ JSON `{ stdout, stderr, exit_code, duration_ms }`.
10. `Backend` → `Browser`: HTTP-ответ с результатом.
11. `Browser` отображает результат в интерфейсе.

На основе этого сценария создайте UML-диаграмму последовательности.

## 4. Детали реализации backend

- Приложение FastAPI (`backend/main.py`):
  - эндпоинт `/` — проверка, что сервис запущен;
  - эндпоинт `/health` — проверяет доступность Docker;
  - эндпоинт `/execute` — принимает Python-код, запускает выполнение в контейнере и возвращает результат.
- Интеграция с Docker:
  - клиент создаётся через `docker.from_env()` на старте приложения;
  - используется образ `pysandbox-executor` (настраивается через переменную окружения `EXECUTOR_IMAGE`);
  - контейнер создаётся с параметрами:
    - `network_disabled=True`;
    - `mem_limit=128m` (или значение из `MEMORY_LIMIT`);
    - `nano_cpus=500000000` (примерно 0.5 CPU, значение настраивается);
    - `remove=True` (auto-remove контейнеров после завершения).
- Ограничение по времени:
  - при ожидании завершения контейнера используется таймаут (`EXECUTION_TIMEOUT_SECONDS`);
  - при превышении таймаута контейнер принудительно завершается.

## 5. Детали реализации frontend

- Страница `index.html`:
  - текстовое поле (textarea) для ввода Python-кода;
  - кнопка «Запустить код»;
  - блоки для вывода `stdout`, `stderr`, кода завершения и времени выполнения.
- Скрипт `script.js`:
  - отправляет запрос `POST /execute` на `http://localhost:8000/execute`;
  - обрабатывает ответ и отображает результат;
  - выводит пользовательские сообщения об ошибках (например, если backend недоступен).
- Стили `styles.css`:
  - оформляют тёмную тему и современный UI.

## 6. Деплой и окружения

- **Локальная разработка**:
  - используется `docker-compose up --build`;
  - фронтенд доступен по `http://localhost:3000`;
  - бэкенд — по `http://localhost:8000`.
- **Продакшн / демонстрация** (по согласованию):
  - можно использовать те же Docker-образы;
  - рекомендуется включить HTTPS и внешнюю аутентификацию при необходимости.
