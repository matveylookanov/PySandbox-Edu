# Техническое задание на разработку системы «PySandbox Edu»

## 1. Общие сведения

- **Наименование системы**: PySandbox Edu.
- **Заказчик**: преподаватель / учебное заведение.
- **Исполнитель**: команда студентов (см. `README.md`).

## 2. Назначение разработки

Создание учебного веб-приложения, демонстрирующего интеграцию транслятора (интерпретатора Python) с Docker для безопасного выполнения пользовательского кода.

## 3. Требования к системе

### 3.1. Функциональные требования

1. Backend на базе FastAPI:
   - реализует эндпоинты `/`, `/health`, `/execute`;
   - использует Docker SDK для запуска контейнеров с пользовательским кодом;
   - применяет ограничения по памяти, CPU и таймауту;
   - логирует запросы и результаты выполнения.

2. Frontend:
   - простой интерфейс без фреймворков (чистый HTML/CSS/JS);
   - поле ввода для кода, кнопка «Запустить», область вывода результата;
   - взаимодействие с API через `fetch`.

3. Executor:
   - отдельный Docker-образ с Python 3.10;
   - запуск кода внутри непривилегированного контейнера;
   - отсутствие сетевого доступа.

4. Docker-compose:
   - поднимает три сервиса: `backend`, `frontend`, `executor`;
   - монтирует сокет Docker в backend.

### 3.2. Нефункциональные требования

- Кроссплатформенность за счёт использования Docker.
- Простота развёртывания (`docker-compose up --build`).
- Полная документация по СТПО, ТЗ, архитектуре и тестированию.

## 4. Требования к составу и параметрам технических средств

- Машина с установленным Docker и docker-compose.
- Рекомендуемые характеристики: не менее 4 GB RAM, 2 CPU.

## 5. Требования к программному обеспечению

- ОС: Windows / Linux / macOS.
- Установленный Docker Engine и docker-compose.
- Git для версионирования.

## 6. Стадии и этапы разработки

1. Анализ требований и проектирование архитектуры.
2. Разработка backend (FastAPI + Docker SDK).
3. Разработка frontend (HTML/CSS/JS).
4. Подготовка Dockerfile для backend, frontend, executor.
5. Написание docker-compose.yml.
6. Написание и заполнение документации.
7. Разработка и проведение тестов.
8. Подготовка отчётов, баг-репортов и презентации.

## 7. Порядок контроля и приёмки

- Проверка структуры проекта на соответствие заданию.
- Проверка работоспособности команды:

  ```bash
  docker-compose up --build
  ```

- Тестирование API:
  - успешный запуск простого кода;
  - обработка ошибок синтаксиса;
  - проверка таймаута;
  - проверка ограничения длины кода.

- Оценка полноты документации и качества тест-кейсов.
