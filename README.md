# PySandbox Edu

Учебный проект — онлайн-песочница для безопасного запуска Python-кода в изолированных Docker-контейнерах.

## Команда проекта

- ФИО 1 — роль
- ФИО 2 — роль
- ФИО 3 — роль

## Архитектура (кратко)

- **frontend (nginx + HTML/CSS/JS)** — веб-интерфейс песочницы (порт 3000).
- **backend (FastAPI)** — REST API для запуска кода и управления исполняемыми контейнерами (порт 8000).
- **executor (Python 3.10)** — минимальный образ, в котором выполняется пользовательский код.

Весь пользовательский код выполняется в отдельных контейнерах на основе образа `pysandbox-executor`. Доступ к Docker осуществляется из backend через `docker.sock`, при этом:

- код запускается с лимитом памяти 128 MB;
- ограничение CPU (примерно 0.5 CPU);
- сеть отключена (`network_disabled=True`).

Подробнее см. `docs/Архитектура.md`.

## Требования

- Docker и docker-compose;
- Git;
- Браузер (Chrome/Firefox/Edge).

## Запуск проекта

1. Клонировать/скопировать репозиторий (после инициализации Git):

   ```bash
   git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
   cd pysandbox-edu
   ```

2. Собрать и запустить сервисы:

   ```bash
   docker-compose up --build
   ```

3. Открыть в браузере:

- frontend: `http://localhost:3000`
- backend: `http://localhost:8000/docs` (Swagger UI)

## API

- `GET /` — проверка, что backend работает.
- `GET /health` — проверка состояния сервера и доступности Docker.
- `POST /execute` — выполнение Python-кода.

Пример запроса:

```bash
curl -X POST "http://localhost:8000/execute" ^
  -H "Content-Type: application/json" ^
  -d "{\"code\": \"print('Hello from PySandbox')\"}"
```

Ответ:

```json
{
  "stdout": "Hello from PySandbox\n",
  "stderr": "",
  "exit_code": 0,
  "duration_ms": 25
}
```

## Безопасность и ограничения

- Код выполняется **только** в контейнере из образа `pysandbox-executor`.
- Для каждого запроса создаётся **новый контейнер** с автоматическим удалением (`remove=True`).
- Ограничения:
  - максимум 10 000 символов кода;
  - 128 MB памяти (настраивается через `MEMORY_LIMIT`);
  - ограничение CPU (через `NANO_CPUS`);
  - отключённая сеть (`network_disabled=True`).
